Configuration

This document describes the files used to configure the JSerWeb server.


Main configuration file

JSerWeb is configured by placing directives in plain text configuration files. The default configuration file is called serweb.conf. The location of this file is '/conf', but you can change the name of this file with the -s command line flag. In addition, other configuration files may be added using other command line flags.

JSerWeb configuration file contain one directive per line. The server will ignore the lines that begin with "#", it considers them comments. You can not include a comment after a configuration directive. Blank lines and white space occurring before a directive are ignored, so you may indent directives for clarity.


Main configuration file directives
Port
Allows you to bind JSerWeb to specific port, instead of the default.

Default: Port 80
ServerName Gives the name that the server uses to identify itself.

Default: ServerName localhost
ServerAdmin Your address, where problems with the server should be e-mailed.

Default: ServerAdmin admin@domain.com
Timeout The number of seconds before receives and send time out.

Default: Timeout 30
KeepAlive

Whether or not to allow persistent connections (more than one request per connection). Set to "off" to deactivate.

Default: KeepAlive On
DocumentRoot The directory out of which you will serve your documents.

Default: DocumentRoot ./httpdoc
ServerRoot The top of the directory tree under which the server's configuration, error, and log files are kept.

Default: ServerRoot ./
LogRoot The location of the log files. In this directory will be saved all the errors (error.log). If you define an errorlog file in a <VirtualHost> container, it will be saved in this directory, too. In the other hand, the access file ('access.log') will be saved here, too. It is not possible to specify an access information will be stored in this file.

Default: LogRoot ./logs
ErrorRoot The directory out of which you will serve your customizable error responses. If you want to change the name of some of the files in this directory, you should modify the file 'conf/errors.conf'

Default: ErrorRoot ./httperrors
TempRoot The top of the directory tree under which the server's configuration, error, and log files are kept.

Default: ServerRoot ./
DirectoryIndex

Sets the file that JSerWeb will serve if a directory is required.

IMPORTANT: Only ONE index file must be specified ni a DirectoryIndex directive.
Default: DirectoryIndex index.html

Plugins
This version of JSerWeb has two plugins, Perl and PHP. To add a new plugin extension you must specify the new extension and the program who will execute this script.

Sintax: LoadPlugin ext program_path

Plugins
LoadPlugin php php.exe
 Define the path to the executable for PHP scripts.
LoadPlugin pl perl.exe
 Define the path to the executable for Perl scripts.
