Virtual Hosts


The term Virtual Host refers to the practice of running more than one web site (such as 127.0.0.3 and 127.0.0.10) on a single machine.

If you want to maintain multiple hostnames on your machine you can setup VirtualHost containers for them. JSerWeb Virtual hosts are "IP-based", it means that you have different IP address for every web site.


Configuring Virtual Hosts

To use multiple virtual hosts you must create a VirtualHost block for each different host that you would like to serve. These blocks must be added into the file conf/vhosts.conf.

For example:

<VirtualHost 127.0.0.2
  ServerName www.dominio2.com.ar
  ServerAdmin postmaster@dominio2.com.ar
  DocumentRoot d:/serweb10/httpdoc/phpmyadmin
  ErrorLog dominio2_com_ar.log
VirtualHost>

<VirtualHost 127.0.0.3
  ServerName www.test.com.ar
  ServerAdmin postmaster@test.com.ar
  DocumentRoot d:/serweb10/httpdoc/test
  ErrorLog test_com_ar.log
VirtualHost>

This example shows how to configure JSerWeb to serve two web sites. The fist one will be available at IP 127.0.0.2 and will serve the files stored at d:/serweb10/httpdoc/phpmyadmin. The second one will be available at IP 127.0.0.3 and will serve the files stored at d:/serweb10/httpdoc/.

Virtual Hosts Directives

These are the directives you can use to configure a Virtual Host.

Virtual Host configuration file directives
VirtualHost
Allows you to specify the web site IP. Also, it defines the Virtual Host block beginning which ends with the directive VirtualHost>

Example:
<VirtualHost 127.0.0.2
  ...
VirtualHost>
ServerName Gives the name that the server uses to identify itself.

Example: ServerName www.domain.com
ServerAdmin Your address, where problems with the server should be e-mailed.

Example: ServerAdmin admin@domain.com
ErrorLog This directive sets the name of the file to which the server will log any errors it encounters. JSerWeb will log the errors in the file /logs/error.log even if you set the log file with this directive. All the log files are saved in the directory /logs.

Example: domain_com.log
DocumentRoot The directory out of which you will serve your documents.
